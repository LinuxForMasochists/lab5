1. Зробіть форк репозиторію на GitHub.
2. Клонуйте форк собі локально через SSH.
3. Створіть нову гілку для своєї зміни:
   - назва гілки повинна описувати суть зміни (наприклад, fix-typo або feature-readme).
4. Робіть атомарні коміти з інформативними повідомленнями.
5. Переконайтесь, що проєкт збирається/працює після змін.
6. Відкрийте Pull Request у вихідний репозиторій з описом змін.

## Стиль комітів

- Повідомлення коміту пишемо українською або англійською, коротко та по суті.
- Приклад: Додано розділ з метою заняття або Add CONTRIBUTING.md.
